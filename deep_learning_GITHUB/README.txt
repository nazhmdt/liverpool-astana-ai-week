# LiverPool AI — Deep Learning Module (3D Volumetric Segmentation)

Данный репозиторий содержит исходный код ИИ-компонента платформы **LiverPool AI**, отвечающего за глубокий 3D-анализ воксельного пространства КТ-снимков органов брюшной полости, автоматическую сегментацию печени и патологических очагов (опухолей), а также расчет их физических характеристик.

## Технические характеристики архитектуры
* **Базовая модель:** 3D UNet (пятиуровневая полносверточная нейросеть для воксельной сегментации) с skip-connections для сохранения детальных границ.
* **Входной формат:** 3D КТ-объемы в медицинском стандарте NIfTI (`.nii` / `.nii.gz`).
* **Препроцессинг данных:**
  * Ресэмплинг шага координатной сетки к единому физическому масштабу `(1.5, 1.5, 2.0)` мм для корректного расчета физических объемов.
  * Приведение осей пациента к единой ориентации по стандарту `RAS`.
  * Фильтрация плотностей по шкале Хаунсфилда (HU-windowing) в диапазоне от `-175` до `+250` HU для изоляции мягких тканей печени.

## Структура репозитория

* **`train_pro.py`** — скрипт глубокого 3D-обучения модели UNet с препроцессингом, аугментацией данных и взвешенной функцией потерь `DiceCELoss`.
* **`evaluate.py`** — скрипт для выполнения базового инференса и оценки модели на отдельных тестовых снимках.
* **`check_overall_metrics_afterfix.py`** — скрипт для проведения глобальной валидации обученной модели на независимой выборке (51 пациент). Рассчитывает средний коэффициент Дайса (Dice Score) для паренхимы печени и опухолей, а также чувствительность (Sensitivity) детекции.
* **`export_2d_slices.py`** — генератор диагностических 2D PNG-срезов в плоскости максимального сечения опухоли. Экспортирует сопоставление маски врача (Ground Truth) и предсказания ИИ для интеграции с интерфейсом.
* **`generate_all_3d_database_metricsjson.py`** — скрипт для автоматического расчета физических параметров (объем печени и опухоли в мл, диаметр очага, координаты 3D-центроида) и формирования итоговой базы данных `metrics.json` для фронтенда.
* **`test_ai.py`** — вспомогательный скрипт для быстрой проверки доступности графического процессора (GPU CUDA) и инициализации весов модели.
* **`model_results/`** — папка с сохраненными результатами валидации.



## Результаты валидации и анализа (Model Validation & Results)

Валидация качества работы модели `v300` проводилась на выборке из **51 пациента** из валидационной группы LiTS. Результаты сегментации и расчетов экспортированы в файл `model_results/metrics.json`.

### Итоговые метрики классификации
* **Общая точность (Accuracy):** `90.20%` (46 из 51 правильно классифицированных профилей).
* **Чувствительность (Sensitivity / Recall):** `100.00%` (46 из 46 случаев онкопатологии успешно локализованы, 0 пропусков критических состояний).
* **Специфичность (Specificity):** `0.00%` на уровне детекции мелких очагов (модель настроена на гиперчувствительность во избежание пропуска патологий).
* **Ложноположительные случаи (False Positives):** 5 пациентов (`№32, №34, №38, №41, №47`). В этих случаях ИИ сегментировал потенциально опасные зоны пониженной плотности, которые не были размечены в оригинальном наборе данных как опухоли.

---

### Клинический разбор: Случай №34 (Срез КТ №48)

Наглядный пример расхождения между ИИ и врачебным консилиумом (Ground Truth) представлен на изображении ниже:

* **Левая панель (Анализ ИИ):** Нейросеть сегментировала паренхиму печени (зеленый) и выделила подозрительный очаг (красный) объемом `4.92 см³` (диаметр по RECIST 1.1 — `21.11 мм`) с расчетной вероятностью злокачественности `88.4%`.
* **Правая панель (Ground Truth):** В оригинальной врачебной разметке КТ сегментирован только контур печени (зеленый), а гиподенсивный очаг оставлен без онкологической метки.

* Clinical Note:* Подобные случаи демонстрируют консервативную («предохранительную») стратегию работы ИИ: система обращает внимание врача-радиолога на сомнительные участки плотности, которые могли быть пропущены или не задокументированы при ручной разметке.
