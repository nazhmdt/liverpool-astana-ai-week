# HepatoGuard AI — рабочий прототип

ИИ-платформа раннего выявления заболеваний печени и хронических вирусных гепатитов.
Кейс №04, Astana AI Week 2026.

Это не просто макет — рабочий код: синтетическая когорта пациентов (калиброванная под
реальную эпидемиологию Казахстана), риск-движок (FIB-4/APRI + XGBoost + SHAP), CV-модель
по снимкам (CNN + Grad-CAM + 3D-реконструкция через marching cubes) и Next.js-интерфейс
на русском языке.

## Структура проекта

```
hepatoguard-ai/
├── backend/     — FastAPI + ML (Python)
│   └── app/
│       ├── data/       — генератор синтетической когорты и снимков
│       ├── models/     — риск-движок, CV-модель, 3D-реконструкция
│       └── main.py     — API-эндпоинты
└── frontend/    — Next.js (TypeScript)
    ├── app/     — страницы (кабинет врача, онко-скрининг, УЗИ/КТ-анализ, аналитика)
    └── components/
```

## Как запустить

Нужны: **Python 3.11+** и **Node.js 20+**.

### 1. Backend

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

pip install -r requirements.txt

# PyTorch ставится отдельно (CPU-версия, легче стандартного pip install torch):
pip install torch --index-url https://download.pytorch.org/whl/cpu
```

Обученные модели и синтетическая когорта уже включены в архив
(`backend/app/data/cohort.pkl`, `backend/app/models/*.pt`, `*.json`), поэтому
можно сразу переходить к запуску сервера. Если хочется пересоздать данные или
переобучить модели заново (например, другой размер когорты) — необязательно:

```bash
python3 app/data/synthetic_generator.py
python3 -m app.models.risk_engine
python3 -m app.models.imaging_engine

# Запустить сервер:
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Backend поднимется на `http://localhost:8000`. Проверить: `curl http://localhost:8000/api/health`
должен вернуть `{"status":"ok","patients_loaded":3000}`.

### 2. Frontend

Открыть **новый терминал**, backend должен продолжать работать:

```bash
cd frontend
npm install
npm run dev
```

Сайт откроется на **`http://localhost:3000`** — это и есть интерфейс, который нужно открыть в браузере.
(`localhost:8000` — это только API, там нет страницы для просмотра, это нормально.)

## Что смотреть в интерфейсе

| Раздел | Что показывает |
|---|---|
| Кабинет врача (`/`) | Список пациентов с авто-расчётом риска по обычным анализам крови |
| Онко-скрининг (`/oncology-screening`) | Все пациенты с ненулевым % онкологического риска, отсортированы по убыванию — кого направлять на углублённое обследование в первую очередь |
| УЗИ/КТ-анализ (`/imaging`) | CV-модель: классификация снимка + Grad-CAM локализация + 3D-реконструкция очага (объём, центроид, вращение/масштаб мышью) |
| Аналитика Минздрава (`/analytics`) | Национальная агрегированная статистика + песочница «что если увеличить охват скринингом» |
| Прозрачность модели (`/model`) | Метрики модели (accuracy, F1, AUC-ROC, матрица ошибок) — честно, с оговорками про синтетические данные |

На карте пациента с высоким/критическим риском есть кнопка «Запустить УЗИ-анализ» —
это и есть цепочка «лаборатория → снимок → подтвердить/исключить онкологию».

## Известные ограничения (честно, для показа жюри/докторам)

- Все данные и снимки **синтетические**, калиброванные под опубликованную статистику
  Казахстана — это не реальные пациенты. Явно указано на странице «Прозрачность модели».
- Bounding box на снимке — это Grad-CAM (локализация по градиенту классификатора), а не
  отдельно обученный детектор — так и написано в интерфейсе.
- Следующий шаг перед реальным пилотом: ретроспективная валидация на деидентифицированных
  исторических данных 2-3 клиник-добровольцев.

## Если что-то не работает

- **Backend не отвечает** — проверить, что оба терминала (backend и frontend) открыты и не упали;
  `curl http://localhost:8000/api/health` должен ответить.
- **3D-реконструкция не вращается / чёрный экран** — убедиться, что вкладка браузера активна
  и видима (не свёрнута, не в фоне) — WebGL не рендерится в неактивных вкладках.
- **Порт занят** — если 3000 или 8000 уже используются, остановить процесс на этом порту или
  указать другой (`uvicorn app.main:app --port 8001`, и поменять `NEXT_PUBLIC_API_URL` в
  `frontend/.env.local` соответственно).
